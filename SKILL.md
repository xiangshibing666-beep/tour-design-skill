# 旅游方案设计 Skill（Tour Design Skill）

## 用途

将一次「多日多城家庭 / 休闲旅游」需求，转化为一套可直接使用、可分享的交付物：

1. **详细行程方案 MD** —— 含全部标准板块（见 `references/tour-plan-template.md`），含【1.4 天气与穿搭总览】章节
2. **网页版 HTML** —— 炫彩、含目录导航、彩色表格，浏览器查看
3. **A4 分享版 PDF** —— 干净打印版，手机 / 微信直接看
4. **交互式行程地图 HTML** —— 高德底图 + 按天切换 + POI 标注 + 路线（见 `assets/map_template.html`）
5. **旅游安全性分析** —— 温度 / 降雨 / 地质灾害 / 风俗习惯（见 `references/safety-analysis-guide.md`）

## 触发条件

用户提出旅游规划、行程规划、旅游方案、攻略、旅游安全分析、行程地图等需求时调用。典型句式：

- 「帮我规划 X 天 Y 城的旅游」
- 「做一份 Z 日游详细方案」
- 「去 XX 那几天天气 / 暴雨安全吗」
- 「给我做个行程地图」

## 工作流程

### 阶段 0 · 需求澄清与约束锁定

先确认以下要素（缺失则主动询问，但不要过度打扰）：

- 目的地城市（1–N 城）与城市顺序（用 WebSearch 在小红书 / 抖音 / 携程 / 去哪儿 consensus 决定最顺环线，避免绕路）
- 出发地与返程地、出发 / 返程日期、总天数
- 出行人数与关系（决定房型、汇合点）
- 出行风格（休闲 / 打卡 / 深度）与硬性约束（如某天门诊、必须午休）
- 已有参考攻略附件（如有，先 Read 解析，标注不合理处）

### 阶段 1 · 多源攻略调研（WebSearch）

并行检索以下主题，每条结论标注来源与日期：

- 城市 / 景点必去清单与「择选」项
- 各景点门票、预约渠道、提前天数、开放时间、闭馆日（**重点核查周一闭馆、限流秒空**）
- 本地人推荐餐厅（老字号 / 网红 / 人均 / 排队提醒）
- 住宿区域与具体酒店（经济 / 舒适 / 品质三档，近地铁）
- 城际与城内交通（高铁班次、车站、耗时；地铁支付方式）
- 目的地 7 日 / 15 日天气预报与历史气候（温度、降雨、台风）
- 当地风俗习惯、禁忌、节庆（用于安全分析的风俗部分）
- 常见旅游陷阱与避坑点

> **金句铁律**：所有数据（票价、预约天数、班次、天气）必须来自检索或官方，未核实标注 ⚠️ 待确认，绝不凭推理编造。

### 阶段 1.5 · 关键事实联网核查（生成方案前必做）

> **目的**：阶段 1 是「广撒网调研」，本阶段是「定稿前的正确性二遍过」——把写进方案里的每一个**硬事实**逐项拉到最新日期联网核实，发现错误当场修正。这是 v4 定稿时补上的环节，专门消灭「凭记忆/凭旧帖写错票价、闭馆日、放票时间」这类问题。

写方案 MD 之前，对以下每类事实做一次**定向核验**（WebSearch / WebFetch 官方或权威源，取 2026.7 当月数据），逐项记录「旧值 → 核实值（来源 + 日期）」，发现不一致立即改：

1. **闭馆 / 限流类**：各博物馆、陵墓、景区的**闭馆日**（重点查周一是否闭馆）、**预约提前天数**、**放票精确时间**（如中山陵提前 7 天 16:00 放票、南京博物院提前 7 天 18:00 放票）。易错点：① 把「人民广场老馆」的闭馆日套到「东馆」头上；② **景点是否「每周 X 闭园」**（如豫园每周一闭园——凡排进行程的景点必须核对其周闭园日，绝不把该景点排在闭园日！2026.7.8 真实踩坑：原方案把豫园排在周一导致进不去，已前移修正）；③ 暑期限定活动是否与其所在景点同闭园日冲突（如豫园夏日奇幻夜周一随本园不开放）。
2. **门票价格**：逐项核实官方票价，**注意淡旺季差价**（如豫园 7 月属淡季 ¥30）、套票、半价政策。易错点：凭旧帖写错（如总统府写成 ¥40，实为 ¥35）。
3. **预约渠道真实性**：公众号 / 小程序名称是否准确、是否还需第三方平台（携程 / 美团）、是否必须实名分时预约。
4. **活动真实性**：暑期限定 / 节庆活动（如豫园夏日奇幻夜、杭州奇妙夜）是否真实存在、时间区间、票价——避免把引流帖杜撰成行程亮点。
5. **交通班次**：航班 / 高铁的**时刻、车站、票价**（标注「以 12306 / 航司实时为准」）；机场选择建议（优先近市区者）。
6. **餐厅存续**：高赞推荐店是否仍在营业、有无搬迁 / 闭店、人均与排队是否仍准确。
7. **天气刷新**：出发前抓取最新 15 天预报，更新逐日温度 / 降雨（尤其后段 Day 5+ 波动最大）；同时抓实时预警（暴雨 / 地灾），用于【1.4 天气与穿搭总览】与安全性分析。

> 核查结论汇总进方案【行程优化 / 修正说明】的「v 最终版核查修正清单」，让用户一眼看到改了什么。

#### 1.5.1 闭馆日 × 日历对照表（必做，消灭"排进闭园日"遗漏）

> **根因**：v5 / v6 连续两次把景点排在周一闭园日（豫园、上海中心），根因是"被动响应式核查"——只在用户质疑时才查。本表改为**一次性系统扫描**：把"每个景点的周闭园日"与"本次行程的具体日历"做一次笛卡尔积核对，凡命中闭园日者一律移出或换备选。

**执行步骤**：

1. 列出本次行程纳入的**全部景点 / 场馆 / 暑期限定活动**（含博物馆、陵墓、园林、观光厅、夜游），逐个查官方周闭园日（无闭园日的标注"每日开"）。
2. 列出行程逐日对应**星期几**（如 7/11 周六、7/13 周一……），生成"日期 → 星期"映射表。
3. 做对照：对每个景点，检查其被排进的那一天是否为其闭园日。命中即标记冲突。
4. 冲突处理优先级：① 平移到其他非闭园日（如豫园周一闭→前移周日）；② 无空间则换同类型备选（如上海中心118层周一疑似闭馆→改朵云书院52层免费）；③ 仍不行则标注"可选·注意周一闭馆"并给 Plan B。
5. 暑期限定活动须与其**所在景点同闭园日**一并核查（如豫园夏日奇幻夜随本园周一不开放）。

**对照表示例（华东三城真实数据）**：

| 景点 | 周闭园日 | 排入日 | 星期 | 冲突？ | 处理 |
|------|----------|--------|------|--------|------|
| 豫园 | 周一（法定假除外） | 7/12 | 周日 | 否 | — |
| 豫园夏日奇幻夜 | 随本园周一不开放 | 7/12 | 周日 | 否 | — |
| 上海中心118层观光厅 | 两源冲突（本地宝称周一不开放 vs 携程称全年开放） | 7/13 | 周一 | 疑似 | **以实际购票验证为准**：用户在官方渠道实测购得7/13周一当天票→确认周一开放，恢复正式行程。方法论：当官方信息冲突时，最硬证据是"能否实际下单购票"，优先采信可购票结果 |
| 上博东馆 | 周二闭（周一开） | 7/13 | 周一 | 否 | — |
| 总统府 | 周一闭 | 7/17 | 周五 | 否 | — |
| 灵隐寺 | 每日开（但免票须提前1天约） | 7/15 | 周三 | 否 | — |

> 此方法固化后，任何新行程只要在阶段 1.5 跑一遍对照表，即可在写方案前彻底排除"景点进不去"类硬伤。

### 阶段 2 · 行程编排

- 生成 N 天行程概览表（日期 / 城市 / 主题 / 住宿地）
- 每天编排【时间轴】表格：精确到 30 分钟，含活动 / 详情 / 交通耗时
- 每天强制预留 12:30–14:00 午休 1.5–2h（避暑 / 休整）
- 每天 1–2 段自由闲逛时间
- 标注需要预约的景点（名称 / 提前天数 / 渠道 / 建议时段）
- 多出发地时做三人汇合方案（城市 + 时间 + 地点）
- 计算总预算（交通 + 住宿 + 餐饮 + 门票 + 其他），人均拆分
- 拍照打卡点（每城 2–3 个机位 + 时段）、避坑指南（每城 3–4 条）、应急信息（电话 / 医院）

### 阶段 2.1 · 景区适宜度多维评估（基于真实游客评价与攻略数据）⭐

不要把排程简化为"凉时户外 / 热时室内 / 傍晚渡江 / 入夜看灯"的单一时间维度——要结合**联网检索的游客评价、攻略实测、官方运营数据**，为每个景点建立"适宜度画像"，再从画像倒推排程。本 skill 的华东案例曾因此踩坑：轮渡被排到 18:30，但东金线**末班 18:15**，联网核实后才卡准窗口。

**8 维适宜度指标**（每景点逐维标注，来源见下）：

| 维度 | 关注点 | 华东真实示例 |
|------|--------|-------------|
| 1. 光线与摄影（Photo） | 黄金时刻（日出/日落前1h）、蓝调时刻（日落后30min）、顺光/逆光、梧桐光影 | 外滩夜景蓝调≈19:30、武康路清晨柔光、上海之巅日落18:55最佳 |
| 2. 人流密度与时序（Crowd） | 高峰（周末正午/节假日）、低谷（工作日上午/清晨） | 武康路9:30后旅行团峰、南京路节假日20万+、外滩夜景人挤但值得 |
| 3. 体感微气候（Thermal） | 户外暴晒/室内空调/江风树荫 | 南京路无遮荫硬伤、东馆强避暑、轮渡江风凉、武康路梧桐稍遮 |
| 4. 运营时间硬约束（Window） | 末班/闭馆日/亮灯时间 | ⚠️ 东金线轮渡末班18:15、豫园周一闭、外滩亮灯≈19:00 |
| 5. 体验时长与节奏（Duration） | 整块 vs 碎片时间 | 东馆需3–4h整块、上海之巅1–1.5h、轮渡15min、武康路1.5–2h |
| 6. 预约/门槛成本（Friction） | 免约/预约/购票/限流 | 东馆免约刷证、上海之巅已购票直进、豫园分时预约 |
| 7. 趣味/情感/亲子（Enjoyment） | 用户偏好、亲子趣味 | 老婆力荐轮渡、南京路MM豆/乐高旗舰店亲子强 |
| 8. 交通接驳成本（Transit） | 地铁直达/换乘/步行 | 金陵东路渡口近外滩核心、东馆地铁2号线上海科技馆站 |

**排程决策优先级（硬约束优先于舒适偏好）**：
1. **运营时间硬约束**先排除不可行时段（轮渡必须18:15前、闭馆日不选）；
2. **体感**把高温段（11:00–15:00）排室内/遮荫（东馆避暑、南京路商场）；
3. **光线**把摄影点排黄金/蓝调（外滩蓝调、武康路清晨、上海之巅日落）；
4. **人流**避开高峰（武康路清晨、南京路上午）；
5. **时长**保证整块时间（东馆下午3h）；
6. **趣味/偏好**嵌入用户钟爱项（轮渡作"桥"而非赶路）。

**资料来源（必须联网，不得凭印象）**：携程/马蜂窝攻略、小红书/抖音实拍、大众点评 UGC、官方公众号/网站、气象景观指数（如上海之巅购票处电子屏"晚霞概率"提示）。凡"末班/闭馆/亮灯"等硬约束，务必联网核实原文。

**排程后必查**：把时间轴每段对照景点适宜度画像，确认"时段↔指标"匹配（轮渡是否卡末班前、外滩是否在蓝调、东馆是否覆盖正午高温）。

### 阶段 3 · 写出方案 MD

按 `references/tour-plan-template.md` 的完整骨架填充，确保包含全部 10 个板块（目录、优化说明、基本信息、每日行程、预算、预约汇总、物品清单、避坑、拍照点、应急信息）。其中【1.4 天气与穿搭总览】使用阶段 1.5 刷新的最新预报填写，给出逐日温度 / 降雨表 + 穿搭总原则，并在每日【穿搭建议】中标注雨天备案。**【1.5 景点适宜度画像（排程依据）】为必含章节**：在阶段 2.1 完成的 8 维画像，必须按模板以"每城一张表"形式落进方案 MD 的 1.5 节，并标注"本方案排程"列——让交付物本身透明、可溯源，不再只是 skill 内部方法论。**排程后必查**：时间轴每段对照 1.5 画像表，确认时段↔指标匹配（轮渡卡末班前、外滩在蓝调、东馆覆盖正午高温）。

### 阶段 4 · 生成三件套（HTML + PDF）

使用 `scripts/build_html_pdf.py` 从 MD 生成网页版 HTML 与 A4 PDF：

```bash
python build_html_pdf.py --md "华东三城8日游详细方案.md" --name "华东三城8日游详细方案" --outdir "."
```

脚本逻辑：markdown 库转换 → 注入炫彩网页 CSS（含 TOC 锚点导航）→ 写 `.html`；注入 A4 打印 CSS → 写 `_pdf.html` → 调用 Chrome / Edge headless 打印 PDF（找不到浏览器则跳过并打印提示，由用户浏览器手动打印）。

### 阶段 5 · 生成交互式行程地图

复制 `assets/map_template.html`，将每日 POI（经纬度、类型、时间、备注）填入 `daysData` 数组（模板变量名固定为 `daysData`）。标注图例：🏨 住宿 / 📍 景点 / 🍽 餐饮 / 🚄 交通。底部 Day1–N 标签一键切换，键盘 1–8 切天，Esc 复位。高德底图国内秒开，失败自动回退 OSM。

> **一致性铁律**：地图上的每个 POI 必须与最终版方案 MD 的每日行程逐条对应——凡方案因闭馆日/天气/用户反馈做过平移或替换（如 Day3 上海中心→朵云书院），地图必须同步修改，不得遗留旧版标注。每次方案升版后，先 diff 方案 MD 与地图 POI 再交付。

### 阶段 6 · 生成旅游安全性分析

按 `references/safety-analysis-guide.md` 方法，结合「既往气候 / 灾害报告」+「未来逐日预报」+「实时预警」+「风俗习惯」，输出独立分析文档（MD + 经 `build_html_pdf.py` 生成的 PDF）。包含：逐城市逐日温度 / 降雨表、各景点暴雨 / 地质灾害脆弱性风险矩阵、台风 / 汛期影响评估、风俗习惯注意事项、综合安全结论 + 分景点 Plan B + 装备清单。

### 阶段 7 · 交付与提示

用 `present_files` 按优先级 **PDF > HTML > MD > 地图 > 安全分析** 呈现。口头总结 3 个最关键提示（头号预约风险、天气风险、行程亮点）。

## 铁律

- **不编撰**：票价 / 班次 / 预约天数 / 天气均须核实，未核实标 ⚠️
- **不推倒重来**：有参考附件时在其基础上优化，明确指出不合理处
- **诚实标注不确定性**：8–15 天预报波动大，提示出发前 3 天复查
