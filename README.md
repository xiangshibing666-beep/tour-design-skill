# tour-design-skill

WorkBuddy 旅游方案设计 Skill —— 把一次「多日多城家庭 / 休闲旅游」需求，转化为一套可直接使用、可分享的交付物。

## 它能做什么

输入「帮规划 X 天 Y 城旅游」类需求，自动产出：

1. **详细行程方案 MD** —— 含分钟级时间轴、餐厅 / 酒店推荐、天气穿搭、预约说明、预算、避坑、拍照点、应急信息，含【天气与穿搭总览】章节
2. **网页版 HTML** —— 炫彩、目录导航、彩色表格，浏览器查看
3. **A4 分享版 PDF** —— 干净打印版，微信 / 手机直接看
4. **交互式行程地图 HTML** —— 高德底图 + 按天切换 + POI 标注 + 路线
5. **旅游安全性分析** —— 温度 / 降雨 / 地质灾害 / 风俗习惯

## 核心特性

- **关键事实联网核查（阶段 1.5）**：写方案前对票价、闭馆日、预约天数、放票时间、天气逐项联网核实，杜绝凭记忆写错
- **闭馆日 × 日历对照表（1.5.1）**：把每个景点的周闭园日与本次行程日历做一次系统扫描，凡命中闭园日者自动平移或换备选——彻底消灭「景点排在闭园日进不去」类硬伤
- **地图一致性铁律**：地图 POI 必须与方案 MD 逐条对应，升版先 diff 再交付

## 安装到 WorkBuddy

将本仓库克隆 / 复制到 WorkBuddy 的 user-level skills 目录：

```bash
# 方式一：克隆
git clone https://github.com/<your-username>/tour-design-skill.git \
  "$HOME/.workbuddy/skills/tour-design-skill"

# 方式二：直接拷贝文件夹到上述路径
```

重启 WorkBuddy 或刷新技能列表后，对助手说「帮我规划 X 天 Y 城旅游」即可触发。

## 使用

对话示例：

> 帮我详细规划华东三城（上海→杭州→南京）7 晚 8 天家庭游，老婆同行，周六出发，要求每日精确到分钟、含午休、餐厅酒店推荐、天气穿搭、预约说明、拍照点、避坑、预算。

技能会按 8 阶段工作流（需求澄清 → 多源调研 → 关键事实核查 → 行程编排 → 写 MD → 三件套 → 地图 → 安全分析）自动产出全部交付物。

## 工作流概览

| 阶段 | 内容 |
|------|------|
| 0 | 需求澄清与约束锁定 |
| 1 | 多源攻略调研（WebSearch） |
| 1.5 | 关键事实联网核查 + 闭馆日×日历对照表 |
| 2 | 行程编排（时间轴 / 午休 / 预约 / 预算） |
| 3 | 写出方案 MD |
| 4 | 生成三件套（HTML + PDF） |
| 5 | 生成交互式行程地图 |
| 6 | 生成旅游安全性分析 |
| 7 | 交付与提示 |

## 依赖

- Python 3（生成 PDF 需本机 Chrome / Edge，headless 打印；缺浏览器时降级为手动打印提示）
- 联网（WebSearch 核查 + 高德地图底图）
